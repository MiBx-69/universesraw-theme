Security Policy

Thank you for helping keep UNIVERSES RAW Shopify Theme and its users secure.

We take security seriously and appreciate responsible disclosure of potential vulnerabilities.

Supported Versions

The following versions are currently supported with security updates.

Version	Supported
3.x.x	✅
2.x.x	❌
1.x.x	❌

Only the latest major release receives security updates.

Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Instead, report it privately using one of the following methods:

Email: mib.bappi360@gmail.com
GitHub Security Advisories: Use the repository's Report a vulnerability feature (if enabled).

Please include:

A detailed description of the issue
Steps to reproduce the vulnerability
A proof of concept (if applicable)
Potential impact
Suggested remediation (optional)

The more information you provide, the faster we can investigate and resolve the issue.

Response Timeline

Our goal is to:

Acknowledge reports within 3 business days
Provide an initial assessment within 7 business days
Keep reporters informed throughout the investigation
Release a fix as soon as reasonably possible

Response times may vary depending on the severity and complexity of the issue.

Scope

This policy applies to the source code contained in this repository.

Examples of issues that may qualify include:

Cross-Site Scripting (XSS)
Injection vulnerabilities
Authentication or authorization flaws
Remote code execution
Sensitive information exposure
Dependency vulnerabilities
Configuration errors that could compromise security
Out of Scope

The following are generally considered out of scope unless they present a clear security impact:

Code style or formatting issues
Minor UI or UX bugs
Browser compatibility issues
Performance improvements
Feature requests
Documentation errors
Responsible Disclosure

Please allow us a reasonable amount of time to investigate and remediate reported vulnerabilities before making them public.

We ask that you:

Do not publicly disclose the issue before a fix is available.
Avoid accessing or modifying data that does not belong to you.
Avoid disrupting services or degrading the experience of other users.
Act in good faith while testing and reporting vulnerabilities.
Security Best Practices

If you use or contribute to this project:

Keep Shopify themes and apps up to date.
Regularly audit project dependencies.
Never commit API keys, access tokens, passwords, or other secrets.
Use environment variables for sensitive configuration.
Enable two-factor authentication (2FA) on your GitHub and Shopify accounts.
Review pull requests carefully before merging.
Disclosure Policy

After a vulnerability has been fixed, we may publish a security advisory describing:

The affected versions
The impact
The resolution
Upgrade recommendations
